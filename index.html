Introduction to the BLINKER pipeline

BLINKER and associated tools are an automated pipeline for detecting BLINKS in EEG and calculating various properties of these blinks. pipeline is a standardized early-stage EEG processing pipeline that focuses on the identification of bad channels and the calculation of a robust average reference. PREP also has an extensive reporting facility. It is designed to be run in a completely automated way. The major sections of this document are:

  • Introduction (requirements, citing, installation)
  • Algorithm (steps, meaning of parameters for each step)
  • Running as an EEGLAB plug-in
  • Running as a script

Requirements

The BLINKER pipeline relies on the MATLAB Signal Processing toolbox and EEGLAB, a freely-available MATLAB toolbox for processing EEG. EEGLAB is available from http://scn.ucsd.edu/eeglab. BLINKER assumes that the EEG data is provided as an EEGLAB EEG structure and that channel locations are provided in the EEG.chanlocs structure.

Installation

The BLINKER pipeline can be run in two ways --- as a standalone toolbox or as an EEGLAB plugin. To run in standalone mode, simply download the BLINKER code, found in the EEG-Blinks github repository and run the script runBlinkerPipeline after editing the locations of your input files.

BLINKER as an EEGLAB plugin

You can install BLINKER as an EEGLAB plugin by unzipping the BlinkerVxx directory into the plugins directory of your EEGLAB installation.
